# North Star — 项目目标与偏离检测

## 唯一目标

**让 AI 协作的质量随时间可度量地提升。**

不是"做更多功能"，不是"更多 Agent"，不是"更复杂的架构"。
是：上周 AI 帮你做的事情，这周做得更好了——有数据证明。

## 三个核心指标

### 1. 反馈覆盖率
定义: 有人类反馈的 significant 任务 / 全部 significant 任务
目标: > 50%
当前: 2% (58 条 trace, 1 条有反馈)
为什么重要: 没有反馈，系统就是瞎的，所有进化都无从谈起

### 2. 任务完成率
定义: completion_status = completed 的 trace / 全部 trace
目标: > 80%
当前: 未知 (只有 30/58 条有 completion_status)
为什么重要: 这是最基本的"AI 在帮忙还是在添乱"的指标

### 3. 人类修改比例 (Post-Edit Ratio)
定义: agent 产出被人类修改的比例
目标: < 15% (改得越少说明 agent 越靠谱)
当前: 未采集
为什么重要: 这是质量的真实度量，不靠人打分

## 偏离检测

每周问自己三个问题:

1. **这周的三个指标比上周好了还是差了?**
   如果连续 2 周没改善 → 系统没在进化，检查 nightly eval 是否在跑

2. **这周加的新功能，有助于提升这三个指标吗?**
   如果答案是"不确定" → 可能在做不必要的功能

3. **这周有没有人类反馈被采纳并改进了 profile?**
   如果连续 2 周没有 → 进化闭环断了

## 什么时候该加新功能 vs 什么时候该停下来用

### 该停下来用的信号
- trace 数据不到 100 条
- golden examples 不到 5 个
- 反馈覆盖率 < 30%
- 还没跑过一次完整的 nightly eval

### 该加新功能的信号
- 某个指标连续 2 周没有改善，且瓶颈明确
- nightly eval 产出了 proposal 但现有系统无法执行
- 用户明确说"这里不好用"

## 当前阶段 (Phase 0 → Phase 1 过渡)

Phase 0 (基建) 已完成。接下来不是继续加功能，而是:

1. **正常工作 1-2 周**，让 trace 自然积累
2. **每天花 30 秒给 1-3 个 significant 任务反馈** (飞书点链接)
3. **积累到 5 个 golden examples**
4. **跑第一次 nightly eval，看 insight 是否有价值**

只有当以上 4 步完成后，才考虑 Phase 1 的功能开发。
