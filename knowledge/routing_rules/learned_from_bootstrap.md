# Routing Rules — 从创建过程学到的

## 任务类型识别信号

### 跨端任务信号
- 用户提到 "iOS + 后端" / "前端 + API"
- 目标涉及两个不同技术栈的文件
- 需要定义新的 API 接口

### 数据分析 vs 内容生成
- "分析" "对比" "统计" → data_agent
- "写文案" "生成报告" "周报" → 目前没有 content_agent，暂由 data_agent 处理
  - 用户反馈: data_agent 做内容生成效果差 (预期中)
  - 触发条件: 如果 data_agent 在内容生成任务上连续 3 次 score < 0.5 → 提议新增 content_agent

### 系统设计类任务
- "调研" "架构" "方案" → orchestrator 直接处理，不分发
- "搭建" "创建项目" → infra_agent + 对应技术栈 agent

## 路由错误的早期信号
- 用户说 "不是这样" 或 "我要的是..." → 可能路由到了错误的 agent
- 用户手动重写了大部分输出 → agent 能力不匹配
- 追问超过 3 次 → 目标理解错误或 agent 能力不足
